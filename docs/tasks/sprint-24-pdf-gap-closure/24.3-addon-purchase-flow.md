# 24.3 - Addon Purchase Flow

Status: TODO. Owner: domain-engineer + event-integration. Spec refs: FR-09 (addon orders), FR-22
(addon/VAS invoice lines), PDF event storming AddonPurchased. Design: design-note.md D1/D3/D4.

## Tasks

1. New event `addon.purchased.v1`: canonical `addon-purchased.avsc` (orderId, customerId,
   subscriptionId, addonCode, addonName, addonType, price, currency, quantity, nullable
   allowance deltas, occurredAt); aggregate_type `addon` -> `addon.events` topic (EventRouter
   routes by aggregate_type - no connector change); pom `<subjects>` entry; compat test;
   event-catalog row (producer order-service; consumers usage-service, billing-service).
2. order-service: fulfillment publishes one event per ADDON item via outbox (bundled orders:
   subscriptionId from activation payload; standalone: target_subscription_id). Standalone saga:
   `PaymentCompletedEventConsumer` branches orderType == ADDON -> confirm + fulfill (no
   activation leg). Audit entries per existing pattern.
3. usage-service: `Quota.addAllowance(minutes, sms, mb)` (raises totals + remaining, clears
   notified flags when back under limits); `TopUpQuotaCommand`/handler with InboxBehavior;
   `AddonPurchasedEventConsumer` (own groupId, eventType-header filter).
4. billing-service: `V2__addon_charges.sql` -> `addon_charge_record` (subscription_id,
   customer_id, addon_code, addon_name, price, currency, purchased_at, billed, invoice_id);
   `AddonPurchasedBillingConsumer` (own groupId, inbox); `BillRunBatchProcessor` adds one
   `InvoiceLine.of(invoice, "Addon: <name> (<code>)", 1, price)` per unbilled charge and marks
   billed inside the bill-run tx.
5. web-bff: `OnboardingCompositionService.placeOrder` forwards `addonCodes` as ADDON items
   (frontend already sends them - zero frontend changes).

## Acceptance criteria

- Bundled and standalone addon purchases produce exactly one CONFIRMED quota top-up and exactly
  one invoice line each (idempotent under Kafka redelivery - tests prove it).
- Onboarding with an addon selected in the real UI reaches FULFILLED with topped-up quota.
