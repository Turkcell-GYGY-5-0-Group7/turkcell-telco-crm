## Tasks

1. `V2__customer_contact.sql`: nullable `email varchar(255)`, `phone varchar(32)` on `customers`.
2. `Customer` entity fields + `updateProfile` extension; `RegisterCustomerRequest` /
   `UpdateCustomerRequest` gain optional `email` (`@Email`) and `phone` (E.164-ish regex), both
   `@Sensitive`-masked in logs; `CustomerResponse` exposes them; commands/handlers threaded.
3. Additive nullable-default-null `email`/`phone` fields on `customer-registered.avsc` and
   `customer-updated.avsc` + compat tests + event-catalog payload docs.
4. Class-level `@ValidIdentityForType` constraint on `RegisterCustomerRequest`: INDIVIDUAL ->
   TCKN checksum, CORPORATE -> VKN checksum (validators already exist, currently unwired for
   VKN); remove the hard-coded field-level `@ValidTckn`. Corporate beyond registration stays out.
5. Update `docs/api-contracts/customer-service.md`; optionally surface email/phone in web-bff
   `ProfileSummary`.

## Acceptance criteria

- Register/update with email/phone persists and returns them; logs mask both.
- INDIVIDUAL + invalid TCKN -> 400; CORPORATE + valid VKN -> 201; CORPORATE + TCKN -> 400.
- Avro compat tests green; customer-service suite green.
