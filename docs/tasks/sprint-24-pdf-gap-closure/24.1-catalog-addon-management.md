# 24.1 - Catalog Addon Management

Status: DONE. Owner: domain-engineer. Spec refs: FR-05 (addon/VAS hierarchy), Scope-In "Admin
paneli icin urun katalog CRUD", the documented-but-unimplemented `POST /api/v1/addons` (known 500).

## Tasks

1. `V2__addon_management.sql`: nullable allowance columns on `addons` (`data_mb bigint`,
   `voice_minutes bigint`, `sms_count bigint`); seed addons (DATA_5GB, DATA_10GB, SMS_500,
   VOICE_300, VAS_CALLERTUNE) and `tariff_addons` links so `GET /api/v1/addons?tariffCode=`
   returns data on a fresh stack.
2. `CreateAddonCommand` + handler (mirrors `CreateTariffCommand`); request carries
   `applicableTariffCodes`; handler attaches via the owning `Tariff.addons` side (join table is
   owned by Tariff - rows never persist otherwise).
3. Wire `POST /api/v1/addons` (ADMIN) into `AddonController`; extend `AddonResponse` with
   allowances; `Addon` entity gains a factory + allowance fields (stays otherwise immutable).
4. `AddonInternalController`: tokenless `GET /internal/addons/{code}/snapshot` (mirrors
   `TariffInternalController`) returning id/code/name/type/price/currency/allowances for
   order-service pricing.
5. Align `docs/api-contracts/product-catalog-service.md` with the implemented shapes.

## Acceptance criteria

- `POST /api/v1/addons` creates an addon linked to the given tariffs and returns 201 (500 gone).
- Fresh stack: `GET /api/v1/addons?tariffCode=<seeded>` returns seeded addons with allowances.
- Internal snapshot reachable in-network, 404 through the gateway (internal-deny), like tariffs.
- Unit tests: handler, join persistence, validation.
