- ADR-022 Frontend and BFF Strategy

## Subtasks

#### 16.2.1 Initialize the SvelteKit project
- ID: 16.2.1
- Title: Scaffold `frontend/web/` with SvelteKit (Svelte 5), TypeScript, and Vite
- Description: Create the frontend workspace at the repository root, separate from the
  `microservices/` Maven reactor, per ADR-022's repository layout decision. Configure TypeScript,
  ESLint/Prettier (or the project's equivalent lint/format tooling), and `.env`-based configuration
  for the BFF/gateway base URL and the Keycloak realm/client settings. App shell only - no business
  pages yet.
- Business Purpose: Establishes the frontend runtime and toolchain the rest of Sprint 16 builds on
  (ADR-022).
- Inputs: ADR-022 (framework decision, repository layout).
- Outputs: `frontend/web/` (SvelteKit project: `package.json`, `svelte.config.js`, `vite.config.ts`,
  `tsconfig.json`, `src/app.html`, `src/routes/+layout.svelte`), `frontend/web/.env.example`.
- Acceptance Criteria:
  - `npm install && npm run build` succeeds inside `frontend/web/` and produces a deployable build.
  - `npm run dev` serves the app shell at `http://localhost:3000` (the `telco-web` Keycloak redirect
    origin).
- Dependencies: none
- Complexity: M

#### 16.2.2 Base routing, layout, and API client wrapper
- ID: 16.2.2
- Title: Add route shells and a typed API client wrapper
- Description: Add SvelteKit routes for `/login`, `/onboarding`, `/account`, `/invoices`
  (placeholder pages), a shared layout (nav, auth-state placeholder), and a typed `fetch` wrapper
  (`src/lib/api/client.ts`) that targets the BFF (`/bff/v1/**`) by default, per ADR-022 ("the
  browser never calls a domain service directly"). Per ADR-022's thin-slice allowance, the wrapper
  MAY fall back to the gateway (`/api/v1/**`) for endpoints not yet composed by the BFF, but never
  targets a domain-service host/port directly.
- Business Purpose: Gives 16.3-16.5 a routing skeleton and a single, swappable place to attach the
  bearer token to outbound calls.
- Inputs: `docs/api-contracts/web-bff.md`, ADR-022.
- Outputs: `frontend/web/src/routes/login/+page.svelte`,
  `frontend/web/src/routes/onboarding/+page.svelte`,
  `frontend/web/src/routes/account/+page.svelte`,
  `frontend/web/src/routes/invoices/+page.svelte`, `frontend/web/src/routes/+layout.svelte`,
  `frontend/web/src/lib/api/client.ts`.
- Acceptance Criteria:
  - All four routes render a placeholder page under the shared layout.
  - The API client wrapper compiles, is unit-testable, and is the only module in the codebase that
    constructs outbound URLs (no ad hoc `fetch` calls scattered across pages).
- Dependencies: 16.2.1
- Complexity: M

#### 16.2.3 CI build/lint stage for frontend/web
- ID: 16.2.3
- Title: Add a Node CI stage for `frontend/web`
- Description: Extend CI (the pipeline built in Sprint 15.4) with a Node/npm job that installs,
  lints, type-checks, and builds `frontend/web` on changes under that path, mirroring the
  build-tests-gate pattern used for the Maven reactor (NFR-17).
- Business Purpose: Keeps the new frontend held to the same CI quality gate as the rest of the
  platform (ADR-014, NFR-17).
- Inputs: Sprint 15.4 CI pipeline, `frontend/web/package.json` scripts.
- Outputs: CI workflow update (new Node job/stage), path-filtered to `frontend/web/**`.
- Acceptance Criteria:
  - A change under `frontend/web/` triggers lint, type-check, and build in CI; the job fails the
    pipeline on lint/type/build errors.
  - Changes outside `frontend/web/` do not trigger the Node job (path filter).
- Dependencies: 16.2.1
- Complexity: S
