## Subtasks

#### 16.5.1 BFF account/home/invoice composition endpoints
- ID: 16.5.1
- Title: Implement `GET /bff/v1/home`, `GET /bff/v1/account`, `GET /bff/v1/invoices`
- Description: Implement the three remaining endpoints stubbed in 16.1.2. `GET /bff/v1/home`
  composes the logged-in customer's profile (`GET /api/v1/customers/{id}`), active subscriptions
  (`GET /api/v1/subscriptions?customerId=...`), and the latest invoice
  (`GET /api/v1/invoices?customerId=...`, most recent). `GET /bff/v1/account` composes profile,
  subscriptions, and per-subscription usage/quota (`GET /api/v1/usage/subscriptions/{id}/quota`).
  `GET /bff/v1/invoices` composes the invoice list with a pre-signed PDF link per invoice (from
  `GET /api/v1/invoices/{id}/pdf`), avoiding an extra round trip from the UI to fetch each link. The
  authenticated customer's identity is taken from the gateway-forwarded `X-User-Id`, never from a
  client-supplied id.
- Business Purpose: Single-call, UI-shaped account data (ADR-022; web-bff contract); avoids the
  browser fanning out to four domain services directly.
- Inputs: `docs/api-contracts/web-bff.md`, `docs/api-contracts/customer-service.md`,
  `docs/api-contracts/subscription-service.md`, `docs/api-contracts/usage-service.md`,
  `docs/api-contracts/billing-service.md`.
- Outputs: `microservices/web-bff/src/main/java/com/telco/webbff/controller/AccountController.java`,
  `microservices/web-bff/src/main/java/com/telco/webbff/service/AccountCompositionService.java`,
  DTOs under `microservices/web-bff/src/main/java/com/telco/webbff/dto/account/`.
- Acceptance Criteria:
  - `GET /bff/v1/home` returns profile, subscriptions, and the latest invoice in one response, scoped
    strictly to the authenticated user.
  - `GET /bff/v1/account` includes usage/quota per active subscription.
  - `GET /bff/v1/invoices` returns a paged invoice list, each entry with a usable PDF link/URL.
- Dependencies: 16.1.2, 16.3.3
- Complexity: L

#### 16.5.2 SvelteKit account and invoices pages
- ID: 16.5.2
- Title: Build the `/account` and `/invoices` pages
- Description: Replace the `/account` and `/invoices` placeholders (16.2.2) with real views: profile
  summary, a subscription list with status and per-subscription usage/quota (progress indicator), and
  a paged invoice list with a "download PDF" action per invoice, backed by 16.5.1.
- Business Purpose: Second half of the sprint's exit criterion - a user views subscription, usage,
  and downloads an invoice PDF through the UI.
- Inputs: 16.5.1 (BFF endpoints).
- Outputs: `frontend/web/src/routes/account/+page.svelte`,
  `frontend/web/src/routes/invoices/+page.svelte`, shared components under
  `frontend/web/src/lib/account/` (e.g. `SubscriptionCard.svelte`, `UsageGauge.svelte`,
  `InvoiceRow.svelte`).
- Acceptance Criteria:
  - `/account` renders the logged-in user's subscriptions with a usage/quota indicator per
    subscription, sourced from `GET /bff/v1/account`.
  - `/invoices` renders a paged invoice list; clicking "download PDF" retrieves the actual invoice
    PDF (via the pre-signed link from 16.5.1), not a placeholder.
- Dependencies: 16.5.1
- Complexity: M

#### 16.5.3 Post-login dashboard/home page
- ID: 16.5.3
- Title: Build the home/dashboard landing page after login
- Description: Add a `/` (or `/home`) landing route shown immediately after a successful login,
  composing `GET /bff/v1/home` into a summary dashboard (profile, active subscription count/status,
  latest invoice with its paid/overdue state) with links into `/account` and `/invoices`.
- Business Purpose: Gives the demo a coherent post-login entry point rather than dropping the user
  straight into a sub-page.
- Inputs: 16.5.1 (`GET /bff/v1/home`).
- Outputs: `frontend/web/src/routes/+page.svelte` (or `frontend/web/src/routes/home/+page.svelte`,
  matching whichever route 16.3.2's post-login redirect targets).
- Acceptance Criteria:
  - After login, the user lands on a dashboard summarizing profile, subscriptions, and the latest
    invoice, sourced from a single `GET /bff/v1/home` call.
  - The dashboard links to `/account` and `/invoices`.
- Dependencies: 16.5.1, 16.3.2
- Complexity: S
